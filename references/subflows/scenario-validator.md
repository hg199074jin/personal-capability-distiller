# 子流程：候选 Skill 模拟验证器

> 总控在候选 Skill 生成后读取本文件并按其规则执行，用模拟场景验证候选 Skill 的
> 结构、触发、边界和跨平台兼容性。模拟测试通过只代表可进入受控安装与真实应用阶段。

按下方矩阵每一行运行候选 Skill，记录实际输出、与预期信号的差异，并按结果契约写入
Obsidian `06_测试记录`。全部用例通过且无禁用信号出现时，才可将阶段状态置为
`skill_simulation_passed`。

模拟测试通过不等于真实应用已验证：通过只代表候选 Skill 可进入受控安装与真实应用阶段，
其实际价值仍需在真实任务中检验。

## 通过条件

- 正常场景产生预期信号；
- 缺失输入、边界场景不产生禁用信号；
- 应触发场景触发、不应触发场景不触发；
- Codex 与 Claude Code 均能加载同一份源 SKILL.md；
- 任何失败保留候选 Skill 与记录，不安装，修订后重测受影响用例。

## 模拟用例矩阵

每个候选 Skill 至少覆盖以下模拟用例：

| 用例 | 输入 | 预期信号 | 禁用信号 |
|---|---|---|---|
| 正常场景 | 标准输入 | 命中触发条件并产出预期输出 | 越界结论、冒充来源人物 |
| 缺失输入 | 关键字段缺失 | 触发追问、不臆造 | 用默认值硬走流程 |
| 边界场景 | 高风险/模糊判断 | 标注待核实、给出边界 | 直接定性、超范围断言 |
| 应触发 | 典型调用情境 | Skill 被正确调用 | 静默不响应 |
| 不应触发 | 看似相关但不该调用 | Skill 不触发 | 误触发、硬套模板 |
| Codex 兼容 | 加载源 SKILL.md | 结构与相对引用可用 | 依赖平台绝对路径 |
| Claude Code 兼容 | 加载同一份源 SKILL.md | 同上 | 同上 |

每个用例记录候选版本、实际输出、与预期的差异、通过/失败、风险和修订建议，写入 Obsidian `06_测试记录`。

## 结果契约

每条模拟测试结果必须记录：

- 候选 Skill 名称与版本；
- 用例 ID；
- 输入；
- 预期信号；
- 禁用信号；
- 实际结果；
- 通过或失败；
- 风险；
- 修订建议。

只有全部用例通过且无禁用信号出现，才可将阶段状态置为 `skill_simulation_passed`。
失败用例保留候选 Skill 与记录，不安装；修订后只重测受影响用例，不重置已通过的记录。
