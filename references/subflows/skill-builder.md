# 子流程：候选 Skill 生成器

> 总控在“人用材料定稿版已确认”后读取本文件并按其规则执行，把判断动作、SOP、
> 追问方式或资产生成方法写成规范候选 Skill。产物进入 Obsidian `04_Skill候选`，
> **不得直接安装到任何平台 Skills 目录**。

只有 `human_material_approved` 状态可以生成 A 档候选 Skill。
输出名称、触发条件、不适用场景、输入、执行步骤、输出、追问、约束、验收标准和测试建议。
区分原话、原意、AI 重构和个人应用。
不得冒充来源人物，不得直接安装，不得写入平台 Skills 目录。

## 触发条件

- 总控已确认人用材料定稿版；
- 判断动作、SOP 或资产生成方法已可复用；
- 用户明确要求生成候选 Skill。

## 不适用场景

- 仍处保真还原前阶段；
- 仅需 C 档快速捕获；
- 内容尚未区分来源观点与 AI 重构。

## 验收标准

- 候选 Skill 自包含，仅使用相对引用；
- 触发条件含应触发与不应触发示例；
- 不冒充来源人物，原话标来源、AI 重构明确标记；
- 不得直接安装到任何平台 Skills 目录，产物进入 `04_Skill候选` 待模拟测试。

## 候选 Skill 契约（九个必需部分）

候选 Skill 必须包含：

1. 名称与触发描述；
2. 使用场景和不适用场景；
3. 输入要求；
4. 逐步执行流程；
5. 信息不足时的追问机制；
6. 约束和风险边界；
7. 原话、原意、AI 重构和个人应用的区分；
8. 输出格式与验收标准；
9. 示例调用与测试建议。

成熟度默认为 `draft`，产物进入 Obsidian `04_Skill候选`，不得直接安装到平台 Skills 目录。

## 触发描述质量

触发描述必须同时提供：

- 应触发示例：典型调用场景，正面案例；
- 不应触发示例：看似相关但不该调用的场景，反面案例。

触发描述要写清“在什么情况下调用本 Skill”，而不是“本 Skill 做什么”。

## 来源与人格边界

- 不得冒充老师、作者或专家本人；
- 原话标明来源；AI 重构明确标记；
- 不得大段复制付费资料或生成可替代原课程的内容；
- 对外输出前检查隐私、保密、版权和引用边界；
- 区分原话、原意、AI 重构和个人应用。
