# workflow.md

## A 档顺序
资料清点 → 保真还原 → 对应关系 → 保真版 → 逻辑版 → 使用版 → 定稿版 → 候选 Skill → 模拟测试 → 安装确认 → 真实应用 → 反馈修订 → 内容复盘 → 能力树更新。

前一阶段未确认时不得进入后一阶段。

## 能力域子目录判断（2026-07-06 新增）

阶段产物（能力卡 `03_能力卡片/`、Skill `04_Skill候选/` / `05_Skill已验证/`、测试记录 `06_测试记录/`、应用反馈 `07_应用反馈/`）的具体落点，应由 `obsidian-librarian` 子流程根据"主领域（`primary_domain`）+ 现有能力库结构"自动决定，不再硬编码为顶层单一目录。

具体规则（详见 subflows/obsidian-librarian.md）：

- **能力库已建主领域子目录时**（如 `03_能力卡片/audit/`、`05_Skill已验证/ai/`）：新资产落到对应子目录（如 CA-XXX 在 `audit/`、SK-XXX 在 `ai/`）。
- **能力库仍为顶层扁平结构时**（旧式）：新资产落到顶层目录，向后兼容。
- **跨主领域资产**（如"AI 提效审计"）：主领域字段自动判子目录；`auxiliary_domains` 仅做引用关系，不重复创建。
- **主领域难以判断时**：追问用户而非默认归类。

阶段产出表的所有"路径"列均视为**示意**，最终路径以 obsidian-librarian 输出为准。
